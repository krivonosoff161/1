## 🔴 P0: КРИТИЧЕСКИЕ ПРОБЛЕМЫ (требуют немедленного исправления)

### P0-1: Breakeven отсутствует полностью
| | |
|---|---|
| **Суть** | Нет механизма "при прибыли > X% → SL = entry_price + fee" |
| **Файл** | Все модули управления позицией |
| **Влияние на PnL** | До 1% потерь на сделку — позиция была в +1.5%, закрылась в +0.9% |
| **Решение** | Добавить в TrailingStopLoss.update(): при profit_pct > 0.8% подтягивать SL к entry_price |

### P0-2: BUY/SELL конфликт — оба сигнала отбрасываются
| | |
|---|---|
| **Суть** | При diff ≤ 0.05 между BUY и SELL — оба удаляются вместо выбора сильнейшего |
| **Файл** | `signal_generator.py:2086-2091` |
| **Влияние на PnL** | 5-10% пропущенных сигналов, потенциальная потеря 0.5-1% |
| **Решение** | Выбирать сильнейший с штрафом -20% к strength |

### P0-3: Множественные пути закрытия позиции
| | |
|---|---|
| **Суть** | 6+ разных мест вызывают закрытие, нет единого entrypoint |
| **Файл** | `orchestrator.py` — разные callback'и |
| **Влияние на PnL** | Race conditions, дублирование, до 0.5% на сделку |
| **Решение** | Все закрытия через ExitDecisionCoordinator |

### P0-4: LiquidationProtector — safety_threshold 50% слишком консервативен
| | |
|---|---|
| **Суть** | При leverage 10x закрывает при убытке 5%, слишком рано |
| **Файл** | `liquidation_protector.py:44` |
| **Влияние на PnL** | Преждевременное закрытие, потеря 2-3% на ложных срабатываниях |
| **Решение** | Уменьшить до 20-30% или сделать адаптивным |

### P0-5: PositionSizer DEPRECATED но может использоваться
| | |
|---|---|
| **Суть** | Модуль помечен DEPRECATED с 04.01.2026, но оставлен для "совместимости" |
| **Файл** | `position_sizer.py:1-6` |
| **Влияние на PnL** | До 2% при несоответствии размера позиции |
| **Решение** | Удалить или добавить raise DeprecationWarning |

### P0-6: WS reconnect — race condition
| | |
|---|---|
| **Суть** | При reconnect нет блокировки торговли до восстановления подписок |
| **Файл** | `websocket_manager.py:26-75` |
| **Влияние на PnL** | Торговля на неполных данных, до 0.5% на сделку |
| **Решение** | Добавить флаг `_reconnecting`, блокировать сигналы |

### P0-7: DRIFT_REMOVE без emergency действия
| | |
|---|---|
| **Суть** | Позиция закрылась на бирже, но осталась в реестре — нет агрессивного действия |
| **Файл** | `position_sync.py:333-346` |
| **Влияние на PnL** | "Призрачная" позиция → повторный вход, до 2% убытка |
| **Решение** | Emergency resync всех позиций при DRIFT_REMOVE с убытком |
