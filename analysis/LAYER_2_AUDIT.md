## 🔴 L2-1: BUY/SELL конфликт — при diff ≤ 0.05 оба сигнала отбрасываются

**Severity:** 🔴 Высокая

**Файл:** `src/strategies/scalping/futures/signal_generator.py:2086-2091`

**Root cause:**
- При равной силе сигналов (diff ≤ 0.05) оба отбрасываются
- Нет fallback на выбор сильнейшего с штрафом

**Код:**
```python
if diff <= 0.05:
    logger.warning(
        f"⚡ {symbol}: КОНФЛИКТ BUY({buy_str:.3f}) vs SELL({sell_str:.3f}) — "
        f"сила равная (diff={diff:.3f}), пропускаем оба сигнала"
    )
    base_signals = []  # ← Оба удаляются!
```

**Влияние на PnL:**
- Потеря валидных сигналов при неопределенности рынка
- Оценка: 5-10% пропущенных сигналов, потенциальная потеря прибыли 0.5-1%

**Доказательство из логов:** Поискать "КОНФЛИКТ BUY vs SELL" и "пропускаем оба"

**Предложение:** Вместо отбрасывания — выбирать сильнейший с штрафом -20% к strength

## 🟡 L2-2: Volume ratio — двойная проверка с разными базами

**Severity:** 🟡 Средняя

**Файл:** `src/strategies/scalping/futures/signal_generator.py:6979-6995`

**Root cause:**
- Две разные проверки объема: `vol_sma20` и `avg_volume`
- Непоследовательная логика

**Код:**
```python
# Проверка 1: SMA20
vol_sma20 = sum(c.volume for c in candles[-20:]) / 20
if vol_cur < vol_sma20 * 1.1:
    return []  # Блокировка

# Проверка 2: avg по prev_candles (lookback разный!)
avg_volume = sum(c.volume for c in prev_candles) / len(prev_candles)
if current_candle.volume < avg_volume * detection_values["min_volume_ratio"]:
    return []  # Блокировка
```

**Влияние на PnL:**
- Ложные блокировки при волатильности объема
- Оценка: 3-5% ложно отфильтрованных сигналов

**Предложение:** Унифицировать на одну проверку с EMA объема

## 🟡 L2-3: Same-side reentry — cooldown есть, но не всегда работает

**Severity:** 🟡 Средняя

**Файл:** `src/strategies/scalping/futures/coordinators/signal_coordinator.py` (предполагаемый)

**Root cause:**
- `cooldown_after_loss_minutes` есть в конфиге (3-8 минут)
- Но проверка может быть не в том месте (должна быть перед входом, не в генераторе)

**Код из конфига:**
```python
cooldown_after_loss_minutes=3  # trending
cooldown_after_loss_minutes=5  # ranging  
cooldown_after_loss_minutes=8  # choppy
```

**Влияние на PnL:**
- Возможен повторный вход сразу после убытка
- "Убыток за убытком" в трендовом рынке
- Оценка: до 2% дополнительных убытков

**Предложение:** Убедиться что cooldown проверяется в `signal_coordinator.process_signals()`

## 🟢 L2-4: FilterManager — не все фильтры могут быть инициализированы

**Severity:** 🟢 Низкая

**Файл:** `src/strategies/scalping/futures/signals/filter_manager.py:25-68`

**Root cause:**
- Фильтры устанавливаются через `set_*_filter()` методы
- Нет проверки что все критические фильтры установлены
- Fallback на legacy логику при ошибке

**Код:**
```python
def __init__(self, data_registry=None):
    self.adx_filter = None  # ← None по умолчанию!
    self.volatility_filter = None
    self.mtf_filter = None
    # ...

async def apply_all_filters(...):
    if self.adx_filter:  # ← Проверка есть, но что если None?
        # применяем фильтр
```

**Влияние на PnL:** Минимальное — fallback на старую логику

**Предложение:** Добавить warning если критические фильтры не установлены

## 🟢 L2-7: Warm-up period — неявный, разный для индикаторов

**Severity:** 🟢 Низкая

**Root cause:**
- FastADX: `period * 3` bars replay
- Остальные индикаторы: неявный warm-up
- Нет единого механизма проверки готовности

**Код:**
```python
# FastADX
adx_window = max(self.fast_adx.period * 3, 30)
for candle in candles[-adx_window:]:
    self.fast_adx.update(...)  # Replay
```

**Влияние на PnL:** Минимальное — индикаторы со временем стабилизируются

**Предложение:** Добавить `is_warmed_up()` метод для всех индикаторов
