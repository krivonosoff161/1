# Документация проекта

## Точки входа
- `docs/PROJECT_ROADMAP.md` — единая дорожная карта проекта.
- `docs/development/DOCUMENTATION_STANDARDS.md` — стандарты ведения документации.
- `docs/DOCUMENTATION_AUDIT_2026-02-21.md` — результат аудита и очистки.
- `docs/reports/2026-02/DOCUMENTATION_DEDUPLICATION_2026-02-21.md` — отчет по глубокой дедупликации.
- `docs/reports/2026-02/DOCUMENTATION_DEDUPLICATION_AGGRESSIVE_2026-02-21.md` — отчет по агрессивному проходу.
- `DOCUMENTATION_INDEX.md` — полное оглавление в корне проекта.

## Структура
- `docs/architecture/` — архитектура и технический дизайн.
- `docs/current/` — актуальные рабочие материалы.
- `docs/plans/` — активные и завершенные планы.
- `docs/fixes/` — выполненные исправления.
- `docs/reports/` — отчеты по сессиям и проверкам.
- `docs/analysis/` — аналитика и разбор причин.
- `docs/archive/` — архив исторических и дедуплицированных материалов.

## Правила
- Новые документы не создаются в корне `docs/`.
- Для отчетов использовать формат имени `YYYY-MM-DD_<topic>.md`.
- Дубли и старые версии переносить в `docs/archive/`.
